# Courier API

**Base URL:** `http://localhost:8080`

---

## Эндпоинты для курьера

### 1. Получить профиль курьера

```
GET /couriers/{courierId}
```

**Параметры пути:**

| Параметр    | Тип  | Описание   |
|-------------|------|------------|
| `courierId` | long | ID курьера |

**Ответ `200`:**

```json
{
  "id": 1,
  "name": "Алексей Смирнов",
  "phone": "+79001234567",
  "isAvailable": true,
  "totalDeliveries": 42,
  "totalEarnings": 12600
}
```

> ℹ️ `totalDeliveries` — количество завершённых доставок. `totalEarnings` — заработок (количество доставок ×
> фиксированная цена за заказ).

---

### 2. Обновить профиль курьера

```
PATCH /couriers/{courierId}
```

**Параметры пути:**

| Параметр    | Тип  | Описание   |
|-------------|------|------------|
| `courierId` | long | ID курьера |

Все поля опциональные.

**Тело запроса:**

```json
{
  "name": "Алексей Иванов",
  "phone": "+79009876543"
}
```

**Ответ `200`** — обновлённый профиль курьера

**Ответ `404`:**

```json
"Курьер не найден: 5"
```

---

### 3. Обновить статус доступности

```
PATCH /couriers/{courierId}/availability
```

**Параметры пути:**

| Параметр    | Тип  | Описание   |
|-------------|------|------------|
| `courierId` | long | ID курьера |

**Тело запроса:**

```json
{
  "isAvailable": true
}
```

**Ответ `200`** — обновлённый профиль курьера

> ℹ️ Курьер автоматически становится недоступным (`isAvailable: false`) когда модератор назначает его на заказ. После
> доставки (`DELIVERED`) курьер снова становится доступным автоматически.

---

### 4. Получить историю заказов курьера

```
GET /couriers/{courierId}/orders
```

**Параметры пути:**

| Параметр    | Тип  | Описание   |
|-------------|------|------------|
| `courierId` | long | ID курьера |

**Параметры пагинации (опциональные):**

```
?page=0&size=10&sort=createdAt,desc
```

**Ответ `200`:**

```json
{
  "content": [
    {
      "id": 1,
      "clientId": 3,
      "courierId": 1,
      "moderatorId": 2,
      "status": "DELIVERED",
      "totalPrice": 850,
      "createdAt": "2026-04-24T10:00:00",
      "updatedAt": "2026-04-24T12:00:00",
      "assignedAt": "2026-04-24T10:30:00",
      "dishes": [
        {
          "id": 1,
          "name": "Борщ",
          "price": 350
        }
      ]
    }
  ],
  "totalPages": 1,
  "totalElements": 1,
  "size": 10,
  "number": 0
}
```

---

### 5. Обновить статус заказа

```
PATCH /orders/{orderId}/status
```

**Параметры пути:**

| Параметр  | Тип  | Описание  |
|-----------|------|-----------|
| `orderId` | long | ID заказа |

**Тело запроса:**

```json
{
  "status": "ON_THE_WAY"
}
```

**Допустимые переходы статусов для курьера:**

| Текущий статус     | Новый статус | Действие              |
|--------------------|--------------|-----------------------|
| `COURIER_ASSIGNED` | `ON_THE_WAY` | Курьер забрал заказ   |
| `ON_THE_WAY`       | `DELIVERED`  | Курьер доставил заказ |

**Ответ `200`** — обновлённый заказ

**Ответ `400`:**

```json
"Недопустимый переход статуса: COURIER_ASSIGNED -> DELIVERED"
```

> ℹ️ После статуса `DELIVERED` курьер автоматически становится доступным (`isAvailable: true`).

## Эндпоинты для модератора

### 6. Получить всех доступных курьеров

```
GET /couriers/available
```

Возвращает курьеров у которых `isAvailable: true`.

**Ответ `200`:**

```json
[
  {
    "id": 1,
    "name": "Алексей Смирнов",
    "phone": "+79001234567",
    "isAvailable": true,
    "totalDeliveries": 42,
    "totalEarnings": 12600
  }
]
```

## Эндпоинты для админа

> ⚠️ Эндпоинты ниже доступны только администратору.

### 7. Создать курьера

```
POST /couriers/admin
```

**Тело запроса:**

```json
{
  "login": "courier1",
  "password": "secret123",
  "phone": "+79001234567",
  "name": "Алексей Смирнов"
}
```

**Обязательные поля:** `login`, `password`, `phone`

**Ответ `201`** — созданный профиль курьера

---

### 8. Удалить курьера

```
DELETE /couriers/admin/{courierId}
```

**Параметры пути:**

| Параметр    | Тип  | Описание   |
|-------------|------|------------|
| `courierId` | long | ID курьера |

**Ответ `204`** — курьер удалён, тело пустое

**Ответ `404`:**

```json
"Курьер не найден: 5"
```
