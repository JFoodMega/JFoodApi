# Dishes API

**Base URL:** `http://localhost:8080`

---

## Эндпоинты для клиента

### 1. Получить список блюд

```
GET /dishes
```

Все параметры опциональные — можно комбинировать между собой.

| Параметр      | Тип    | Допустимые значения                                                      |
|---------------|--------|--------------------------------------------------------------------------|
| `cuisineType` | string | `ITALIAN`, `JAPANESE`, `RUSSIAN`, `GEORGIAN`, `AMERICAN`, `OTHER`        |
| `dishType`    | string | `SOUP`, `SALAD`, `MAIN_COURSE`, `APPETIZER`, `DESSERT`, `DRINK`, `OTHER` |
| `name`        | string | любая строка, поиск по вхождению                                         |

**Примеры запросов:**

```
GET /dishes                                    — все активные блюда
GET /dishes?cuisineType=ITALIAN                — итальянская кухня
GET /dishes?dishType=SOUP                      — только супы
GET /dishes?name=борщ                          — поиск по названию
GET /dishes?cuisineType=RUSSIAN&dishType=SOUP  — русские супы
```

**Ответ `200`:**

```json
[
  {
    "id": 1,
    "name": "Борщ",
    "description": "Классический борщ",
    "price": 350,
    "calories": 250,
    "weightVolume": "500г",
    "imageUrl": "https://...",
    "cuisineType": "RUSSIAN",
    "dishType": "SOUP",
    "isActive": true,
    "extras": [
      {
        "id": 1,
        "name": "Сметана",
        "price": 50
      },
      {
        "id": 2,
        "name": "Хлеб",
        "price": 30
      }
    ]
  }
]
```

> ℹ️ Возвращаются только активные блюда (`isActive: true`). Неактивные не отображаются.

---

### 2. Получить блюдо по ID

```
GET /dishes/{dishId}
```

**Параметры пути:**

| Параметр | Тип  | Описание |
|----------|------|----------|
| `dishId` | long | ID блюда |

**Ответ `200`:**

```json
{
  "id": 1,
  "name": "Борщ",
  "description": "Классический борщ",
  "price": 350,
  "calories": 250,
  "weightVolume": "500г",
  "imageUrl": "https://...",
  "cuisineType": "RUSSIAN",
  "dishType": "SOUP",
  "isActive": true,
  "extras": [
    {
      "id": 1,
      "name": "Сметана",
      "price": 50
    },
    {
      "id": 2,
      "name": "Хлеб",
      "price": 30
    }
  ]
}
```

**Ответ `404`:**

```json
"Блюдо с id=5 не найдено"
```

## Эндпоинты для админа

> ⚠️ Все эндпоинты ниже доступны только с ролью администратора.

---

### 3. Создать блюдо

```
POST /dishes/admin
```

**Тело запроса:**

```json
{
  "name": "Борщ",
  "description": "Классический борщ",
  "price": 350,
  "calories": 250,
  "weightVolume": "500г",
  "imageUrl": "https://...",
  "cuisineType": "RUSSIAN",
  "dishType": "SOUP",
  "extras": [
    {
      "name": "Сметана",
      "price": 50
    },
    {
      "name": "Хлеб",
      "price": 30
    }
  ]
}
```

**Обязательные поля:** `name`, `price`, `description`, `weightVolume`, `cuisineType`, `dishType`

**Ответ `201`** — созданный объект блюда (структура та же что в GET)

**Ответ `409`:**

```json
"Блюдо с названием 'Борщ' уже существует"
```

---

### 4. Обновить блюдо

```
PATCH /dishes/admin/{dishId}
```

**Параметры пути:**

| Параметр | Тип  | Описание |
|----------|------|----------|
| `dishId` | long | ID блюда |

Все поля опциональные — передаёшь только те что нужно изменить.

**Тело запроса:**

```json
{
  "name": "Борщ украинский",
  "price": 400,
  "calories": 280,
  "extras": [
    {
      "name": "Сметана",
      "price": 60
    }
  ]
}
```

> ⚠️ Если передаёшь `extras` — список полностью перезаписывается новым. Если `extras` не передаёшь — существующие extras
> не трогаются.

**Ответ `200`** — обновлённый объект блюда

**Ответ `404`:**

```json
"Блюдо с id=5 не найдено"
```

---

### 5. Включить / выключить блюдо

```
PATCH /dishes/admin/{dishId}/status
```

**Параметры пути:**

| Параметр | Тип  | Описание |
|----------|------|----------|
| `dishId` | long | ID блюда |

**Тело запроса:**

```json
{
  "isActive": false
}
```

**Ответ `200`** — обновлённый объект блюда

**Ответ `404`:**

```json
"Блюдо с id=5 не найдено"
```

> ℹ️ Неактивные блюда не отображаются в `GET /dishes` для клиентов.

---

### 6. Удалить блюдо

```
DELETE /dishes/admin/{dishId}
```

**Параметры пути:**

| Параметр | Тип  | Описание |
|----------|------|----------|
| `dishId` | long | ID блюда |

**Ответ `204`** — блюдо удалено, тело ответа пустое

**Ответ `404`:**

```json
"Блюдо с id=5 не найдено"
```

## Справочник значений

### CuisineType

| Значение   | Описание     |
|------------|--------------|
| `ITALIAN`  | Итальянская  |
| `JAPANESE` | Японская     |
| `RUSSIAN`  | Русская      |
| `GEORGIAN` | Грузинская   |
| `AMERICAN` | Американская |
| `OTHER`    | Другое       |

### DishType

| Значение      | Описание       |
|---------------|----------------|
| `SOUP`        | Суп            |
| `SALAD`       | Салат          |
| `MAIN_COURSE` | Основное блюдо |
| `APPETIZER`   | Закуска        |
| `DESSERT`     | Десерт         |
| `DRINK`       | Напиток        |
| `OTHER`       | Другое         |
