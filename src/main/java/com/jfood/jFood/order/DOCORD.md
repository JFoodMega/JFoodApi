# Orders API

**Base URL:** `http://localhost:8080`

---

## Эндпоинты для клиента

### 1. Создать корзину (добавить блюда)

```
POST /orders
```

**Тело запроса:**

```json
{
  "clientId": 1,
  "dishIds": [1, 2, 3]
}
```

**Обязательные поля:** `clientId`, `dishIds`

**Ответ `201`:**

```json
{
  "id": 1,
  "clientId": 1,
  "courierId": null,
  "moderatorId": null,
  "status": "CART",
  "totalPrice": 850,
  "deliveryType": null,
  "deliveryTime": null,
  "addressId": null,
  "createdAt": "2026-04-24T10:00:00",
  "updatedAt": "2026-04-24T10:00:00",
  "assignedAt": null,
  "dishes": [
    {
      "id": 1,
      "name": "Борщ",
      "price": 350
    }
  ]
}
```

> ℹ️ Заказ создаётся со статусом `CART`. Для оформления используй `/checkout`.

---

### 2. Оформить заказ

```
POST /orders/{orderId}/checkout
```

**Параметры пути:**

| Параметр | Тип | Описание |
|---|---|---|
| `orderId` | long | ID заказа (корзины) |

**Тело запроса — как можно скорее:**

```json
{
  "addressId": 1,
  "deliveryType": "ASAP"
}
```

**Тело запроса — к определённому времени:**

```json
{
  "addressId": 1,
  "deliveryType": "SCHEDULED",
  "deliveryTime": "2026-04-24T15:00:00"
}
```

**Обязательные поля:** `addressId`, `deliveryType`

> ⚠️ Если `deliveryType: SCHEDULED` — поле `deliveryTime` обязательно. При `ASAP` время доставки проставляется автоматически (`now + 30 минут`).

**Ответ `200`** — заказ со статусом `CREATED`

**Ответ `400`:**

```json
"Заказ уже оформлен"
```

```json
"Укажите время доставки"
```

**Ответ `404`** — заказ или адрес не найден

### 3. Получить заказ по ID

```
GET /orders/{id}
```

**Параметры пути:**

| Параметр | Тип | Описание |
|---|---|---|
| `id` | long | ID заказа |

**Ответ `200`** — объект заказа

**Ответ `404`:**

```json
"Заказ не найден: 5"
```

### 4. Получить заказы клиента

```
GET /orders/client/{clientId}
```

**Параметры пути:**

| Параметр | Тип | Описание |
|---|---|---|
| `clientId` | long | ID клиента |

**Параметры пагинации (опциональные):**

```
?page=0&size=10&sort=createdAt,desc
```

**Ответ `200`:**

```json
{

  "totalPages": 3,
  "totalElements": 25,
  "size": 10,
  "number": 0
}
```

## Эндпоинты для курьера

### 5. Обновить статус заказа

```
PATCH /orders/{orderId}/status
```

**Параметры пути:**

| Параметр | Тип | Описание |
|---|---|---|
| `orderId` | long | ID заказа |

**Тело запроса:**

```json
{ "status": "ON_THE_WAY" }
```

**Допустимые переходы:**

| Текущий статус | Новый статус | Действие |
|---|---|---|
| `COURIER_ASSIGNED` | `ON_THE_WAY` | Курьер забрал заказ |
| `ON_THE_WAY` | `DELIVERED` | Курьер доставил заказ |
| `COURIER_ASSIGNED` | `CANCELED` | Отменить |
| `CREATED` | `CANCELED` | Отменить |

**Ответ `200`** — обновлённый заказ

**Ответ `400`:**

```json
"Недопустимый переход: ON_THE_WAY -> CREATED"
```

> ℹ️ После статуса `DELIVERED` курьер автоматически становится доступным (`isAvailable: true`).

## Эндпоинты для админа

> ⚠️ Эндпоинты ниже доступны только администратору.

### 6. Получить все заказы

```
GET /orders/admin
```

**Параметры пагинации (опциональные):**

```
?page=0&size=20&sort=createdAt,desc
```

**Ответ `200`** — страница со всеми заказами

---

### 7. Получить заказы по статусу

```
GET /orders/admin/status?status=CREATED
```

**Параметры запроса:**

| Параметр | Тип | Описание |
|---|---|---|
| `status` | string | Статус заказа |

**Примеры:**

```
GET /orders/admin/status?status=CREATED       — новые заказы
GET /orders/admin/status?status=DELIVERED     — завершённые
GET /orders/admin/status?status=CANCELED      — отменённые
```

**Ответ `200`** — страница заказов с указанным статусом

---

### 8. Получить заказы в процессе

```
GET /orders/admin/in-progress
```

Возвращает заказы со статусами `COURIER_ASSIGNED` и `ON_THE_WAY`.

**Ответ `200`** — страница заказов в процессе

---

### 9. Удалить заказ

```
DELETE /orders/admin/{id}
```

**Параметры пути:**

| Параметр | Тип | Описание |
|---|---|---|
| `id` | long | ID заказа |

**Ответ `204`** — заказ удалён, тело пустое

**Ответ `404`:**

```json
"Заказ не найден: 5"
```

## Справочник

### Статусы заказа

| Статус | Описание |
|---|---|
| `CART` | Корзина, ещё не оформлена |
| `CREATED` | Оформлен, ожидает назначения курьера |
| `COURIER_ASSIGNED` | Курьер назначен |
| `ON_THE_WAY` | Курьер в пути |
| `DELIVERED` | Доставлен |
| `CANCELED` | Отменён |

### Типы доставки

| Тип | Описание |
|---|---|
| `ASAP` | Как можно скорее (~30 минут) |
| `SCHEDULED` | К определённому времени |
